Bookinfo Multi-Cluster Architecture

Red Hat Connectivity Link (Kuadrant) + Istio Ambient Mesh + Service Interconnect (Skupper) + ACM GitOps

HTTPS traffic flow
DNS resolution (Route 53)
Kuadrant DNS updates / ACM GitOps
TLS (Let's Encrypt)
Service Interconnect (Skupper)

DNS-Based Traffic Switching

Kuadrant's DNSPolicy on each spoke writes weighted CNAME records to Route 53. Adjusting the weight (0-100) on each spoke shifts traffic between clusters via DNS.

Route 53 Record Chain

bookinfo.spoke.aws-apac.mobb.cloud → klb.bookinfo... → geo-oc.klb.bookinfo... → NLB hostname → IPs. Geolocation + weighted routing.

TLS via Let's Encrypt

TLSPolicy triggers cert-manager to issue certificates via ACME DNS-01 challenge against Route 53. IRSA provides AWS credentials to cert-manager pods.

GitOps via ACM Hub

ApplicationSets on the hub generate ArgoCD Applications per spoke. Switching appsets (e.g. spoke1-appdown) sets weight to 0 and shifts all traffic to the other cluster.

Service Interconnect (Skupper v2)

Skupper Sites on each spoke are linked via AccessGrant/AccessToken. Listeners and Connectors expose productpage, reviews, details, and ratings (port 9080) as virtual services across both clusters. When Skupper is enabled, normal K8s Services are replaced by Skupper-managed routing.
